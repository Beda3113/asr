#DeepSpeech2 

DeepSpeech2 — это архитектура глубокой нейронной сети для задач автоматического распознавания речи (ASR), разработанная компанией Baidu в 2016 году. Она предназначена для преобразования аудиосигнала с речью в текстовую транскрипцию.

Архитектура DeepSpeech2 строится на трёх основных компонентах: свёрточная нейронная сеть (CNN), рекуррентная нейронная сеть (RNN) и выходной линейный слой. Сначала из входного аудиосигнала извлекаются признаки — обычно мел-спектрограмма, которая визуально представляет частоты звука во времени. Затем CNN-экстрактор обрабатывает эту спектрограмму, выделяя ключевые акустические паттерны и одновременно сжимая данные по времени и частоте для уменьшения вычислительной сложности. После этого последовательность сжатых признаков поступает на вход двунаправленных рекуррентных слоёв (обычно GRU или LSTM), которые анализируют контекст как с левой, так и с правой стороны от текущего момента времени. Это позволяет модели лучше понимать связь между звуками в середине фразы. Далее выходы рекуррентных слоёв проходят через полносвязный слой с функцией активации Softmax, который вычисляет вероятности для каждого символа алфавита в каждый момент времени.

Ключевая инновация DeepSpeech2 — использование CTC-функции потерь (Connectionist Temporal Classification). CTC позволяет обучать модель, не требуя точного ручного выравнивания аудио и текста, — сеть сама учится определять, какой звук какому символу соответствует. Благодаря этому обучение становится проще и требует меньше размеченных данных. На этапе декодирования вероятности символов преобразуются в итоговый текст с помощью CTC-декодера. Для улучшения качества распознавания применяется Beam Search — алгоритм, который рассматривает несколько наиболее вероятных вариантов расшифровки одновременно. DeepSpeech2 также поддерживает аугментацию данных (например, добавление шума, изменение тона), что повышает устойчивость модели к различным условиям записи. Архитектура остаётся популярной в академической среде и в промышленных приложениях благодаря балансу между вычислительной эффективностью и качеством распознавания.

### Конфигурация обучения

Модель **DeepSpeech2** содержит **17.7 млн параметров** и обучается с использованием **CTC-функции потерь**. Размер батча установлен в **72** — это максимальное значение, стабильно работающее на V100 16GB без переполнения памяти. Для ускорения загрузки данных задействовано **16 процессов (num_workers)**, что полностью загружает все 24 ядра CPU. В качестве оптимизатора выбран **AdamW** с начальной скоростью обучения **0.0001**.

Мел-спектрограмма вычисляется с **80 фильтрами** и окном **FFT=400** с шагом **160** семплов. Для предотвращения переобучения используется **dropout 0.3** в рекуррентных слоях. Валидация выполняется на датасете **dev-clean** каждую эпоху, а чекпоинты сохраняются в папку `outputs/test_100/`. Все эксперименты логируются в **TensorBoard**, что позволяет отслеживать динамику loss, CER, WER и нормы градиента в реальном времени.
