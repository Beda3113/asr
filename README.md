# asr
# Краткая аннотация к техническому заданию (ТЗ)

## Название работы
Реализация системы автоматического распознавания речи (ASR) на основе архитектуры DeepSpeech2 с CTC-декодированием.

---

## Цель работы
Разработать и обучить модель распознавания речи, способную транскрибировать аудиофайлы в текст, используя библиотеку PyTorch и предложенный шаблон проекта.

---

## Основные требования

| Категория | Требование |
|-----------|-------------|
| **Архитектура** | DeepSpeech2 (CNN + BiGRU + CTC) или LAS |
| **Декодирование** | Hand-crafted Beam Search (собственная реализация, без готовых библиотек) |
| **Аугментации** | Минимум 4 ASR-аугментации (SpecAugment, шумы, изменение темпа и т.д.) |
| **Датасет** | LibriSpeech (train-clean-100/360, dev-clean, test-clean) |
| **Логирование** | W&B / CometML / TensorBoard (loss, gradient norm, lr, аудио, спектрограммы) |
| **Инференс** | `inference.py` с поддержкой пользовательских датасетов |
| **Демо** | Ноутбук для Google Colab (клонирование репозитория, загрузка весов, инференс) |

---

##  Что нужно реализовать

1. **Модель** — DeepSpeech2 с нуля (без готовых реализаций)
2. **Beam Search** — собственная реализация (штраф -20 за использование библиотек)
3. **Аугментации** — 4+ (показать спектрограммы до/после)
4. **`CustomDirDataset`** — для загрузки пользовательских данных
5. **`inference.py`** + **`calc_metrics.py`** — инференс и подсчёт CER/WER
6. **Demo‑ноутбук** — для Google Colab (обязательное требование)
7. **Отчёт** — анализ экспериментов, графики, выводы


## ✅ Ключевые выводы для исполнителя

1. **Код должен быть чистым** — pre-commit, осмысленные коммиты, аккуратный README
2. **LLM запрещены** — всё пишется самостоятельно
3. **Демо в Colab** — обязательно, без него работа не принимается
4. **Beam Search** — только ручная реализация
5. **Аугментации** — 4 штуки с визуализацией спектрограмм

PyTorch Project Template — это готовый промышленный шаблон для тренировки и инференса нейросетей с организацией кода по стандартам ML-инженерии.
Это система распознавания речи (ASR) на основе DeepSpeech2 с CTC-декодированием. Модель принимает на вход аудиофайл и выдаёт текстовую транскрипцию.

## Основные возможности

| Компонент | Назначение |
|-----------|------------|
| **Hydra** | Управление конфигами (YAML) — меняй параметры без правки кода |
| **PyTorch Lightning-подобный Trainer** | Циклы обучения/валидации, логирование, чекпоинты |
| **Абстрактные классы** | `BaseDataset`, `BaseMetric`, `BaseTrainer` — легко расширять |
| **Готовые ASR-компоненты** | CTC Loss, WER/CER, LibriSpeech, Beam Search |
| **Логирование** | W&B, CometML, TensorBoard из коробки |
| **Pre-commit** | Автоформатирование (black, isort) перед каждым коммитом |

## Ключевые файлы

| Файл | Назначение |
|------|------------|
| `train.py` | Запуск обучения |
| `inference.py` | Запуск распознавания |
| `src/trainer/trainer.py` | Логика одного шага (forward, backward, оптимизация) |
| `src/datasets/base_dataset.py` | Загрузка аудио → спектрограмма → текст |
| `src/text_encoder/ctc_text_encoder.py` | Символьная токенизация + Beam Search |
| `configs/` | Все гиперпараметры (модель, lr, датасет, аугментации) |

## Структура модулей

| Папка | Ответственность |
|-------|------------------|
| `datasets/` | Загрузка данных, аугментации, паддинг батчей |
| `model/` | Архитектура нейросети (DeepSpeech2, Baseline) |
| `text_encoder/` | Преобразование текст ↔ числа, CTC/Beam Search декодинг |
| `loss/` | Функция потерь (CTC Loss) |
| `metrics/` | Character Error Rate, Word Error Rate |
| `trainer/` | Цикл обучения, валидация, сохранение чекпоинтов |
| `transforms/` | Аугментации спектрограмм и аудио (SpecAugment) |
| `logger/` | Логирование в W&B/Comet/TensorBoard |
| `utils/` | Вспомогательное: seed, пути, чтение JSON |
