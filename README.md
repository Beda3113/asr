# Реализация системы автоматического распознавания речи (ASR) на основе архитектуры DeepSpeech2 с CTC-декодированием.

---

## Цель работы
Разработать и обучить модель распознавания речи, способную транскрибировать аудиофайлы в текст, используя библиотеку PyTorch и предложенный шаблон проекта.

---

## Основные требования

| Категория | Требование |
|-----------|-------------|
| **Архитектура** | DeepSpeech2 (CNN + BiGRU + CTC) или LAS |
| **Декодирование** | Hand-crafted Beam Search (собственная реализация, без готовых библиотек) |
| **Аугментации** | Минимум 4 ASR-аугментации (SpecAugment, шумы, изменение темпа и т.д.) |
| **Датасет** | LibriSpeech (train-clean-100/360, dev-clean, test-clean) |
| **Логирование** | W&B / CometML / TensorBoard (loss, gradient norm, lr, аудио, спектрограммы) |
| **Инференс** | `inference.py` с поддержкой пользовательских датасетов |
| **Демо** | Ноутбук для Google Colab (клонирование репозитория, загрузка весов, инференс) |

---

##  Что нужно реализовать

1. **Модель** — DeepSpeech2 с нуля (без готовых реализаций)
2. **Beam Search** — собственная реализация (штраф -20 за использование библиотек)
3. **Аугментации** — 4+ (показать спектрограммы до/после)
4. **`CustomDirDataset`** — для загрузки пользовательских данных
5. **`inference.py`** + **`calc_metrics.py`** — инференс и подсчёт CER/WER
6. **Demo‑ноутбук** — для Google Colab (обязательное требование)
7. **Отчёт** — анализ экспериментов, графики, выводы


## Ключевые выводы для исполнителя

1. **Код должен быть чистым** — pre-commit, осмысленные коммиты, аккуратный README
2. **LLM запрещены** — всё пишется самостоятельно
3. **Демо в Colab** — обязательно, без него работа не принимается
4. **Beam Search** — только ручная реализация
5. **Аугментации** — 4 штуки с визуализацией спектрограмм

#  Датасет LibriSpeech

## Общая информация

**LibriSpeech** — это крупный корпус английской речи (~1000 часов), созданный на основе бесплатных аудиокниг из проекта LibriVox. Является стандартным бенчмарком для задач распознавания речи (ASR).

- **Частота дискретизации:** 16 kHz
- **Формат аудио:** .flac
- **Лицензия:** CC BY 4.0
- **Источник:** [OpenSLR](http://www.openslr.org/12/)


## Структура датасета

| Сплит | Размер (архив) | Кол-во примеров | Описание |
|:---|:---|:---|:---|
| **train-clean-100** | 6.3 GB | 28,539 | Обучение (100 часов, чистая речь) |
| **train-clean-360** | 23 GB | 104,014 | Обучение (360 часов, чистая речь) |
| **train-other-500** | 30 GB | 148,688 | Обучение (500 часов, сложная речь) |
| **dev-clean** | 337 MB | 2,703 | Валидация (чистая речь) |
| **dev-other** | 314 MB | 2,864 | Валидация (сложная речь) |
| **test-clean** | 346 MB | 2,620 | Тестирование (чистая речь) |
| **test-other** | 328 MB | 2,939 | Тестирование (сложная речь) |


# Работа с TODO в проекте

## 1. `src/datasets/base_dataset.py`

**Задача:** Применить wave-аугментации до вычисления спектрограммы.

**Решение:**
- Добавлен метод `apply_wave_augs()`
- Изменён `__getitem__`: wave-augs → спектрограмма → spectral-augs

**Результат:** Wave-аугментации (Gain, PitchShift, AddNoise) применяются до спектрограммы, spectral-аугментации (FreqMask, TimeMask) — после.

---

## 2. `src/text_encoder/ctc_text_encoder.py`

**Задача:** Реализовать CTC decode и Beam Search.

**Решение:**
- `ctc_decode()` — удаление blank-символов и повторяющихся токенов
- `ctc_beam_search()` — собственная реализация beam search без внешних библиотек


---

## 3. `src/model/deepspeech2.py`

**Задача:** Создать полную модель DeepSpeech2.

**Решение:**
- CNN экстрактор (2 слоя с Hardtanh)
- 5 слоёв BiGRU с BatchNorm и Dropout
- Линейный классификатор


---

## 4. `src/transforms/` (4+ аугментации)

**Задача:** Реализовать минимум 4 ASR-аугментации.

**Решение:**

- `add_colored_noise.py` — добавление шума (Wave)
- `freq_mask.py` — частотное маскирование (Spectral)
